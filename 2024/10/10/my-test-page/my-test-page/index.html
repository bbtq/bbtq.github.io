2. 构建和运行Cargo项目

2.1 cargo build

  • cargo build

    - 创建可执行文件： target/debug/hello_cargo

  • 第一次运行cargo build 会在顶层目录生成 cargo.lock 文件

    - 该文件负责追踪项目依赖的精确版本

    • 这个lock文件可以帮助该项目，在另外一个环境下，相关依赖的精确配置，保证项目的运行（不像很多项目，在自己的环境中可以运行，但是到了其它地方不能运行，或配置麻烦。）

    - 不需要手动修改该文件

  • 为发布构建

    cargo build –release

    - 生成的可执行文件在 target/release 不再target/debug 里面

2.2 构建和运行cargo项目

  • cargo run，编译代码+执行项目

    - 如果之前编译成功过，并且源码没有改变，那么就会直接运行二进制文件

  • cargo check，检查代码，确保能通过编译，但是不产生任何可执行文件

  • cargo check 比cargo build快得多

    - 要养成良好习惯，编写代码的时候周期性的使用 “ cargo check ”检查代码，提高效率

4.2 引用与借用

在之前的例子里面，函数直接拥有了值的所有权，如果函数结束后不return回去，这个值的所有内容及空间就会在作用域结束后被销毁。
所以有了引用和借用 这样比较优雅的方式来去，让函数借用这个值，而不拥有其所有权。

引用（_reference_）：引用像一个指针，因为它是一个地址，我们可以由此访问储存于该地址的属于其他变量的数据。 与指针不同，引用确保指向某个特定类型的有效值。
借用（_borrowing_）：我们将创建一个引用的行为称为 借用。

引用 用 &
同样和 C/C++ 一样，解引用 用的是 *

数据竞争（_data race_）类似于竞态条件，它可由这三个行为造成：

  • 两个或更多指针同时访问同一数据。
  • 至少有一个指针被用来写入数据。
  • 没有同步数据访问的机制。

6.1.2 枚举方法

6.1.3 Option 枚举与NULL空值

Rust没有空值，但是有一个概念相似的枚举：Option<T>，系统中定义如下，

这个枚举的对于原有的NULL，他是分成了两个部分来代表：“有效”和“无效”这两个概念。

  • Some(T) 包含一个值 T，表示该值存在。
  • None 表示没有值，表示空或者无效。

使用Option<T>的时候，虽然指向了某种类型。但这个Option<T>其实是一种另外的类型。
也就是说，你如果说： let some_num:Option<i32> = None; 或者 let some_num1 = Some(5);但是这个some_num他并不能和一个i32 的值相加，编译的时候就会报错。必须将这个Option<T>中的T转换出来。因为some_num是Option<T>类型的。
这样就避免了类似c语言中，一个值是null，后面这个null没有初始化就直接参与计算，但是却能过编译埋下雷的风险。

使用Option<T>强制要求了人们不能忽略错误处理，类似于if x != null，但Option是强制性的

  • 1.在使用这个Option<T> 时不需要将其显式引入作用域，因为这个 Option<T> 在prelude(预导入模块)中。
  • 2.不用使用 Option:: 这个前缀来使用其中的变体。
    • 可以直接使用：
      • Option<T>
      • Some(T)
      • None

<T> 是泛型类型参数，意味着：

  Option 枚举的 Some 成员可以包含任意类型的数据
  同时每一个用于 T 位置的具体类型使得 Option<T> 整体作为不同的类型

如何使用变体中Some（T）中储存的T值，书中介绍的时候提了一嘴：
想要一些代码只当拥有 Some(T) 值时运行，允许这些代码使用其中的 T。也希望一些代码只在值为 None 时运行，这些代码并没有一个可用的 T 值。match 表达式就是这么一个处理枚举的控制流结构：它会根据枚举的成员运行不同的代码，这些代码可以使用匹配到的值中的数据。

7.使用包、Crate 和模块管理不断增长的项目

“模块系统”（项目结构）包括：

  • |- Packages（包）：Cargo 的一个功能，它允许你构建、测试和分享 crate。
    • |-Crates （单元包）：一个模块的树形结构（模块树），它形成了库或二进制项目。
      • |-Modules（模块）和 use：允许你控制作用域和路径的私有性。
        • |-path（路径）：一个命名例如结构体、函数或模块等项的方式。

7.1 包和 Crate

包就是项目的最顶层，使用cargo new的时候创建的就是一个package和基本的内部组织；

  • Crate（单元包） 有两种类型：
    • binary （二进制的）
    • library （库）

binary(二进制项) 可以被编译为可执行程序，比如一个命令行程序或者一个 web server。它们必须有一个 main 函数来定义当程序被执行的时候所需要做的事情。目前我们所创建的 crate 都是二进制项。
library(库) 并没有 main 函数，它们也不会编译为可执行程序，它们提供一些诸如函数之类的东西，使其他项目也能使用这些东西。比如 第二章 的 rand crate 就提供了生成随机数的东西。大多数时间 Rustaceans 说的 crate 指的都是库，这与其他编程语言中 library 概念一致。

  • Crate root (单元包的根)
    - 是源代码文件
    - Rust编译器从这里开始，组成你的Crate的根模块

    使用 cargo new 创建（Packages）包； 可以看到终端的内容
